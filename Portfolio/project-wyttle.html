Team size: 3 • Timeframe: November 2025 – March 2026 • Role: Project Manager + Backend Lead NDA restrictions are now lifted, so this page includes the engineering work I directly contributed.

Product & engineering detail

Wyttle connects peers-to-peers and peers-to-mentors through guided discovery, matching, and secure in-app communication. I helped deliver the core interaction systems and backend-connected user journeys that support ongoing expansion into university communities.

Stack breakdown

  • Frontend: React Native + Expo with TypeScript for role-specific screens and reusable components.
  • Backend/data: Supabase-backed authentication and data workflows for user identity, matching, and messaging.
  • Application logic: Swipe discovery and match rules connected to real user profiles and role constraints.
  • Communication: Message-thread flows with room creation and message lifecycle actions.

Features I helped ship

  • Sign-in and sign-up flows, including sign-in with Apple, Google, and LinkedIn.
  • Discovery stack for meeting members with swipe mechanics and match logic.
  • Message threads with room creation and support for send/edit/delete message actions.
  • Admin panel for approving join requests, moderating users, and setting mentor prices.

My contribution

I worked as Project Manager + Backend Lead, owning backend planning and integration delivery while helping shape product-critical flows end to end.

Technical challenge

A key challenge was learning Supabase quickly and integrating it properly with the app’s authentication, data access patterns, and user interaction flows.

Outcome

Wyttle currently has 200 members, with daily growth as we continue targeted rollout across university audiences.
